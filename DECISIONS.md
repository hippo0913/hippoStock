# 设计决策记录

## D001
**决策**：使用 Adapter 模式隔离所有外部依赖
**原因**：方便后续替换券商、数据源、通知渠道，core 层零改动
**影响**：所有外部调用必须通过 adapters/ 目录下的抽象接口

## D002
**决策**：单进程架构，不做微服务
**原因**：本地验证阶段，单人开发，无并发压力，优先快速迭代
**影响**：后续如有性能问题再针对性拆分，不提前设计

## D003
**决策**：技术栈选 Python，不混用其他语言
**原因**：akshare、Anthropic SDK、飞书 SDK 均为 Python 原生，开发效率最高；系统瓶颈在网络 IO，语言性能无关紧要
**影响**：所有模块统一 Python 实现

## D004
**决策**：前端用单页 HTML，不用 React 等框架
**原因**：本地自用，单文件更简单，Claude Code 生成质量更稳定
**影响**：前端为单个 HTML 文件，内嵌 CSS 和 JS
